# Contributing to SPARC

Thank you for considering contributing to SPARC! This project aims to make communication more accessible for Deaf and speech-impaired individuals, and every contribution helps bring us closer to that goal.

## 🚀 How to Contribute

### Reporting Bugs

- Use [GitHub Issues](https://github.com/Hazz-Y/SPARC-Smart-Perception-Assistive-Reality-Companion/issues) to report bugs
- Include your Python version, OS, and hardware setup
- Provide steps to reproduce the issue
- Include relevant log output

### Suggesting Features

- Open an issue with the `enhancement` label
- Describe the use case and expected behavior
- Explain why this would benefit the SPARC community

### Code Contributions

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create** a feature branch: `git checkout -b feature/your-feature`
4. **Make** your changes following the code style below
5. **Test** your changes
6. **Commit** with a clear message: `git commit -m 'Add: description'`
7. **Push** to your fork: `git push origin feature/your-feature`
8. **Open** a Pull Request against `main`

## 📝 Code Style

- Follow PEP 8 for Python code
- Keep modules under 300 lines when possible
- Add docstrings to all public functions
- Use type hints where appropriate
- Maintain the existing modular architecture

## 🔬 Areas for Contribution

| Area | Description |
|:---|:---|
| 🔤 Languages | Add support for new sign languages (BSL, JSL, etc.) |
| 🧠 Models | Improve ML model accuracy and inference speed |
| 📱 Mobile | Build a companion mobile application |
| 🌐 Web | Create a monitoring web dashboard |
| 📖 Docs | Improve documentation, tutorials, and examples |
| 🐛 Bugs | Fix bugs and improve stability |
| ⚡ Performance | Optimize inference latency and resource usage |
